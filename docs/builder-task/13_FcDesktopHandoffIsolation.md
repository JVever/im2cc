# 13 /fc 接回电脑后的结果隔离修复

## 背景
- 用户在飞书里接入某个对话后，再在电脑端执行 `fc <名称>` 把该对话接回电脑。
- 预期是：远程连接立即失效，飞书/微信不应再收到该对话的新结果；电脑端也不应继续与一个仍在远程执行的旧任务并发冲突。
- 实际观察到：接回电脑后，飞书仍会收到新的消息，而且内容和电脑端看到的不完全一致，容易误判成两端各自跑了不同任务。

## 根因
1. 本地 `fc` 只归档了远程 binding，没有中断 daemon 内已经在执行的 inflight 任务。
2. 队列在任务开始时把远程 `conversationId` 和 `sendReply` 闭包固定下来；后续即使 binding 已归档，旧任务仍可能继续把流式/最终结果发回原群。
3. daemon 重启恢复 inflight 结果时，只按 `conversationId` 发回，不校验该远程连接是否仍然指向同一 session。

## 方案
- 为 queue 增加“结果回传前绑定校验”：只有当前 conversation 仍然绑定到原 session，才允许发送。
- 为 startup recovery 增加 session 级校验：若远程已解绑或已切换到别的 session，直接丢弃旧 inflight 结果。
- 为本地 `fc` 增加 inflight 中断：接回电脑时主动中断该远程连接仍在执行的任务。
